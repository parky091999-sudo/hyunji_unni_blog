# 워드프레스 심층분석 파이프라인 설계 (v1 — 2026-07-07)

> 워드프레스 확장의 단일 기준 문서. 콘텐츠 포맷·페이지 구성·데이터 소스·파이프라인 흐름을 여기서 관리한다.
> 관련: [WRITING_SYSTEM.md](WRITING_SYSTEM.md)(글쓰기 원칙 — 충돌 시 함께 갱신), `generator/wp_render.py`(렌더러).

---

## 0. 목표와 원칙

- **목표**: 고CPC 정보성 **심층분석** 글을 최종적으로 **완전 자동**으로 워드프레스에 발행 → 구글 애드센스 수익화. (수익화 스위치는 **2026-09-21 이후** — 실업급여 규정.)
- **왜 심층분석(얕은 다량 X)**: 구글 scaled content abuse 정책(2024~)이 저품질 대량생성을 강등(40~45% 감축)하고, AI 오버뷰가 얕은 "X란?" 트래픽을 잠식. 깊은 글은 AI가 인용하는 쪽이고 체류·백링크·에버그린 보상을 받는다.
- **깊이의 정의**: 학술적 난해함·무작정 긴 글이 아니라 **정보 이득(information gain)** — ①원본 종합 ②실데이터·계산 예시 ③비교와 판단 프레임 ④엣지 케이스/흔한 오해 ⑤접근성(현지언니 톤 유지).
- **완전 자동화의 전제**: **숫자는 AI가 짓지 않고 실데이터에서 fetch**(§3). 네이버 파이프라인의 검증 원칙(수치 할루시네이션 금지, 무근거 인용 하드 게이트) 그대로 재사용.
- **반자동은 온램프**: ①품질 바 수립(사람, 가상페이지 검토) → ②실데이터 배선+게이트 → ③완전자동+주기 스팟체크로 수렴.
- **중복 리스크 관리**: 네이버판 얕은 리라이팅 금지 — 같은 주제라도 구글용으로 **실질 재생성**(더 깊게·새 각도·계산 추가). 판단 기준: "네이버 글을 본 사람에게도 새 가치를 주는가".

## 1. 심층분석 포맷 스펙 (v1 — '연금저축 vs IRP' 샘플 승인 기준)

표준 구조(post dict 마커 레벨 → wp_render가 HTML 변환):

| # | 블록 | 내용 규칙 |
|---|---|---|
| 1 | 두괄식 도입 (2~3문단) | 첫 문단에 결론. 둘째 문단에 "이 글이 답할 질문"과 분석 축 예고 |
| 2 | `[요약삽입]` | ✓ 3줄 — 핵심 결론만 (스캔 독자용) |
| 3 | 핵심 수치 스트립 (v2) | `key_stats` 3~4개 — {수치, 라벨} 카드. 글의 뼈대 숫자 |
| 4 | 목차 (v2) | h2에서 자동 생성(앵커 링크). 렌더러가 조립 |
| 5 | 본문 섹션 4~6개 | `[구분선]`+소제목. 각 섹션: **결론 첫 문장 → 근거/계산 예시 → 표·리스트**. 필수 유형: ①구조 해설 ②돈 계산 예시(실수치 ×·= 명시) ③의사결정 프레임("그래서 나는 어떻게") ④흔한 오해/함정 |
| 6 | `[표삽입]` 1개+ | 비교표. 헤더행+첫열 강조(렌더러 CSS) |
| 7 | `[FAQ삽입]` 3~5개 | 검색 롱테일 질문 — FAQPage 스키마 자동 |
| 8 | 출처 (v2) | `sources` 리스트 — 공식기관 위주(E-E-A-T). 본문 수치의 근거 |
| 9 | 면책+저자 (v2) | 금융/세무 면책 박스 + 저자 박스(E-E-A-T 신호) |

- **분량**: 2,500~4,000자 (네이버 1,400~2,000자보다 김 — 구글 심층 의도. 단 §WRITING_SYSTEM 가독성 원칙 유지: 문장 50자·문단 2~3줄).
- **제목**: 검색의도+정보 이득 명시. 예: "연금저축펀드 vs IRP, 세액공제만 보면 손해 봅니다 | 4개 축으로 뜯어본 심층 비교".
- **소제목**: 결론/수치 포함(§WRITING_SYSTEM 5) — "유동성: IRP는 '전부 아니면 전무'".

## 2. 페이지 구성 스펙 (v2 가상페이지 확정 요소)

- 상단: breadcrumb → h1 → 메타라인(**게시일+수정일**(freshness 신호)·읽는 시간·저자)
- 핵심 수치 스트립(카드 3~4) → 요약박스 → 목차 → 본문
- 표는 모바일 가로 스크롤 래퍼(`overflow-x`)
- 하단: 출처 리스트 → 면책 박스 → 저자 박스 → 관련 글 카드 2~3(허브-스포크 내부링크)
- 반응형: 데스크톱 760px 본문 카드 / 모바일 390px 검증 필수
- 검토 워크플로: **호스팅 없이 로컬 HTML 가상페이지**(scratchpad 빌더 → `wp_심층분석_샘플*.html`)로 품질·렌더 검토 후 렌더러에 반영
- **스타일 배포(2026-07-07)**: 승인 v2 CSS를 `poster/wp_assets/hyunji-style.{css,php}`로 이식 → 서버 `wp-content/mu-plugins/`에 배포(테마 독립 mu-plugin, `.hj-*` 스코프). wp_render는 클래스만 출력하고 스타일은 사이트가 가진다. CSS 수정 시 repo 파일 고치고 scp 재배포.

## 3. 실데이터 소스 매핑 (완전 자동화의 팩트 앵커)

| 카테고리 | 핵심 수치 | 소스 | 배선 상태 |
|---|---|---|---|
| 연금·절세 | 세액공제 한도/율, 연금소득세율, 종합과세 분기점 | **연도 갱신 상수**(아래) + FSS finlife 연금저축 공시 | 상수 신설 필요 / FSS 배선됨 |
| 금융재테크 | 예·적금 금리 공시 | FSS finlife (`info_collector._fetch_fss_deposit_rates`) | 배선됨 |
| 주식·ETF·공모주 | 시세·PER·배당·수요예측 경쟁률·확약 | `stock_collector`/`etf_collector`/네이버 IPO+38 | 배선됨(재사용) |
| 보험 | 공시이율·상품 공시·공식 비교도구 | FSS finlife + 보험다모아 안내 | 부분 배선 |
| 부동산 | 기준금리·LTV/DTI 상수, 실거래가 | 연도 상수 + 국토부 실거래 API(추후) | 미배선 |
| 공통 | 최근 뉴스·이슈 | 네이버 뉴스 API(`_fetch_naver_news`) | 배선됨 |

- **연도 갱신 상수 레이어(신설 예정)**: 세율·한도·기준일 같은 제도 수치는 `data/constants_2026.yml`에 모아 **사람이 연 1회(또는 개정 시) 검증** — 자동 생성 글의 팩트 앵커. AI는 이 파일+API 값만 인용 가능.
- 소스에 없는 수치 → 본문에서 "공식 확인" 처리(네이버 파이프라인 `_UNSOURCED_RE` 게이트 재사용).

## 4. 파이프라인 아키텍처 (생성→검증→발행)

```
키워드/주제 선정  ── keyword.py 풀 + 주제 클러스터 쿨다운 재사용 (허브-스포크: 기둥글 주제 우선)
      ↓
리서치 스캐폴딩   ── §3 소스 fetch → facts JSON (상수+API+뉴스)
      ↓
심층 원고 생성    ── §1 구조 프롬프트 + "facts만 사용" 룰(_COMMON_RULES 이식) + 계산 예시 강제
      ↓
품질 게이트       ── quality.py 확장: 기존(무근거 인용·AI상투어·길이) + 심층 지표
      |              (계산 예시 ≥1 · 표 ≥1 · FAQ ≥3 · 섹션 ≥4 · key_stats 유효)
      ↓ (미달 시 피드백 재생성 — 기존 루프 재사용)
렌더              ── wp_render.render_wordpress_post + v2 요소(toc/key_stats/sources 파라미터 추가 예정)
      ↓
발행              ── poster/wp_publish.py (WP REST, 호스팅 후 신규)
      |              온램프: status=draft로 올려 사람 스팟체크 → 안정화 후 publish 자동 전환
      ↓
이력/내부링크     ── 발행 이력 JSON + 같은 클러스터 글 상호 링크(허브-스포크)
```
